# ep011 書き直し後の運用メモ

第十一話「呼吸する部屋」を書き直したあとに、GitHub 上で管理・更新する対象をまとめる。
本文を書き換える前提のメモであり、本文そのものの方向性はここでは決めない。

---

## 基本方針

- ep011 は共通ルートの範囲内なので、編集は必ず `main` ブランチで行う。
- `route/shenzhen` 側で ep011 を直接編集しない。
- `main` で ep011 を確定したあと、`route/shenzhen` へ `main` をマージして共通ルートを同期する。
- ep012 以降の分岐内容は、ep011 の新ラストに合わせて必要最小限だけ見直す。

---

## 差し替え手順

1. `main/011_呼吸する部屋.md` を書き直し版で更新する。
2. `git diff -- main/011_呼吸する部屋.md` で旧版との差分を確認する。
3. 次の観点で ep012 との接続を確認する。
   - ep011 の終わりがパーイ夜のままか
   - シオンがチェンマイへ戻る発言を残すか
   - 発信元不明通知の内容が ep012 末尾の通知と重複しないか
   - 「呼吸」「煙」「音楽」「同期」の重みが ep012 に残りすぎていないか
4. 必要なら `main/012_なんでもない一日.md` の冒頭だけを調整する。
5. 設定・監査資料を更新する。
6. `edit: ep-011 呼吸する部屋 改稿` でコミットする。
7. `route/shenzhen` に `main` をマージして、共通ルート 001〜011 を同期する。

## ep011 改稿で特に守ること

- ep011 は共通ルートの最終話なので、分岐前の「橋」として扱う。
- main 線にも route/shenzhen 線にも使える終わり方にする。
- ep012 以降の具体的な行き先を強く固定しすぎない。
- 発信元不明通知の正体を説明しない。
- 「保存区域」「ノイズ」「同期」を説明しすぎず、身体感覚として残す。

## GitHub 反映メモ

ローカルでコミットしたあと、GitHub へ反映する。

```bash
git push origin main
git checkout route/shenzhen
git merge main
git push origin route/shenzhen
git checkout main
```

`published/ep-011` タグが既にある場合、タグは動かさない。
公開後の大幅改稿として残すなら、必要に応じて `published/ep-011-rev2` を新規作成する。
