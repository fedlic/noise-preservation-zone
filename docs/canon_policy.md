# Canon / Lore 運用ポリシー

このリポジトリは、AI創作用の設定資料として使う。
ただし、すべての未来を先に確定させない。

---

## 基本原則

```yaml
principle:
  canon_source: "本編小説で実際に描写された事実"
  lore_source: "AIが参照してよい世界設定・補助資料"
  future_policy: "未確定の未来は unknown または unresolved として扱う"
  update_rule: "本文に書かれた出来事だけを、あとから設定へ反映する"
```

---

## Canon

Canon は、本編小説で実際に描写された事実だけを指す。

Canon に入るもの:

- 人間関係の進展
- 恋愛関係
- 裏切り
- 結末
- 発信元不明の正体
- キャラクターの重大な秘密
- 本編内で発生した出来事
- 本編内で明言された過去

Canon の扱い:

- 本文にない重大事実を、設定ファイルだけで公式化しない。
- 本文に書かれる前の未来展開は `unknown` または `unresolved` とする。
- Canon を更新するときは、根拠となるエピソード番号を記録する。

---

## Lore

Lore は、AIが参照してよい世界設定・補助資料を指す。

Lore に入るもの:

- HUD
- 発信元不明
- チェンマイ
- パーイ
- 登場人物の既出情報
- 用語
- 年表
- 舞台の雰囲気
- テクノロジーの運用感

Lore の扱い:

- 本文生成の方向性を助ける。
- 未来の確定には使わない。
- 伏線候補や解釈候補は書いてよいが、確定事実と混ぜない。

---

## 未確定情報

未確定情報は、以下の状態で管理する。

```yaml
status:
  unknown: "情報がない"
  unresolved: "匂わせや候補はあるが、本文では未確定"
  canon: "本文に描写され、公式事実として反映済み"
```

---

## PONとシオンの関係

現時点では未確定。

```yaml
relationship:
  pon_shion:
    current_status: unresolved
    canon_relationship: unknown
    future: unknown
```

禁止:

- 将来付き合うと書く
- 将来結婚すると書く
- 将来別れると書く
- 恋愛関係だと設定ファイルで先に確定する
- 本文にない感情の結論を設定資料だけで決める

許可:

- 距離が近い描写を記録する
- 互いに影響を与えている可能性を記録する
- 沈黙、触覚、呼吸、視線、同じ場所にいることを事実として記録する
- 本文に実際に書かれた出来事をあとから反映する

---

## 更新手順

1. まず本編を書く。
2. 本編に描写された事実だけを抽出する。
3. Canon に反映するか、Lore に留めるかを判断する。
4. 未確定の未来は `unknown` または `unresolved` のまま残す。
5. 重大設定は、本文で描写されるまで確定しない。
